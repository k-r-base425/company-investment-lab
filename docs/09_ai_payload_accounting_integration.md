# AI分析JSONへの会計データ反映

## 目的

ホーム画面の「AI分析用データ」コピー時に、保存済みの会計入力データをJSONへ反映します。

これにより、売上・経費・家計・仕訳の入力内容をもとに、事業の収益性、家計支出、経費率、投資可能額をAIへ渡しやすくします。

## 月次サマリーの作成フロー

1. コピー直前に対象月の会計入力データを読み込む
2. 保存済みデータがある場合は、そのデータを使う
3. 保存済みデータが空の場合は、サンプルデータを一時的に使う
4. `calculateMonthlyAccountingSummary` で月次サマリーを計算する
5. `buildAccountingAnalysisPayload` でカテゴリ別・判定別の集計を作る
6. `buildHomeAiAnalysisPayload` でホームAI分析用JSONへ反映する

対象月はMVPでは `2026-06` 固定です。

## business / household / accountingAnalysis の関係

- `business`
  - 保存済み会計データの売上、経費、利益、税金目安、投資可能額、経費率、利益率を入れます。
- `household`
  - 保存済み家計入力の合計支出、固定費、変動費、カテゴリ別支出を入れます。
- `accountingAnalysis`
  - 月次サマリー、カテゴリ別集計、必要支出・浪費・投資の集計、固定費・変動費の集計、直近入力を入れます。

既存の `investment`、`monthlyChart`、`learning` は残します。

## コピー直前に最新データを読み込む理由

会計入力画面で追加・編集・削除されたデータを、ホーム画面に戻った直後でもAI分析JSONへ反映するためです。

ホームカードの表示値はMVPではサンプル中心ですが、コピーされるJSONは保存済み会計データを優先します。

## 今回できること

- 会計入力データの編集
- 編集後の月次サマリー再計算
- AI分析JSONへの保存済み会計データ反映
- Web / GitHub Pagesでは `localStorage` の保存内容を利用
- ネイティブではSQLiteの保存内容を利用

GitHub Pagesは公開URLです。

実データ、個人情報、APIキー、実CSV、SQLite DBは入力・コミットしないでください。
