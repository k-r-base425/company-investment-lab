# 学習メモ検索

## 目的

学習テーマごとのメモが増えたときに、必要なメモをキーワードや分類から探せるようにする。

## メモが増えたときに検索が必要になる理由

学習メモは、会計学習、投資学習、AI分析履歴から蓄積される。件数が増えると、最近のメモだけでは過去の気づきに戻りづらくなるため、検索とフィルターで見返しやすくする。

## キーワード検索の対象

- メモタイトル
- メモ本文
- テーマ名
- カテゴリ
- source
- 関連AI分析タイトル
- 関連画面

大文字小文字は区別しない。日本語は通常の `includes` で判定する。

## カテゴリフィルター

- すべて
- 会計
- 簿記
- 投資
- AI
- 経営
- その他

## sourceフィルター

- すべて
- 手動
- AI分析
- 会計
- 投資

## AI分析由来のみ表示

`source === "ai_analysis"` または `sourceAiAnalysisRunId` があるメモだけを表示する。AI分析履歴から学習メモ化した内容を優先して見返すためのフィルター。

## テーマフィルター

会計学習、投資学習、AI分析結果の見返しにある学習テーマを対象に、テーマ単位でメモを絞り込める。

## 並び替え

- 新しい順
- 古い順
- テーマ順
- カテゴリ順

新しい順と古い順では `updatedAt` を優先し、なければ `createdAt` を使う。

## AI分析JSONとの関係

既存の `learningMemos` サマリーは維持しつつ、手動メモ数、AI分析由来メモ数、カテゴリ別件数を追加する。ホームAI分析JSONやJSONエクスポート側の構造を壊さず、学習メモの振り返りに使える情報を増やす。

- 検索結果からの編集・削除
- 条件リセット
