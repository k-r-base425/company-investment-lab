# 改善効果トラッキング

## 目的

改善アクションの進捗と会計KPIを並べて確認し、行動と数字を一緒に見直せるようにします。

この機能は、改善アクションが数字に与えた可能性を確認するための参考情報です。因果関係は断定しません。

## 対象月

- 内部値：`2026-06`
- 画面表示：2026年6月

## 使用するデータ

- 保存済み会計入力データ
- 保存済み改善アクション

Web / GitHub Pagesでは `localStorage` の保存データを使います。
ネイティブ実行では既存SQLite保存のデータを使います。

## アクション進捗サマリー

`period === "2026-06"` の改善アクションを対象に、以下を計算します。

- 合計アクション数
- 未完了件数
- 完了件数
- 保留件数
- 高優先度の未完了件数
- 高優先度の完了件数
- 達成率
- 高優先度達成率

## 会計KPIスナップショット

既存の `calculateMonthlyAccountingSummary` を使います。

- 売上
- 経費
- 家計支出
- 利益
- 税金目安
- 投資可能額
- 経費率
- 利益率

## 改善進捗インサイト

ルールベースで以下のようなコメントを生成します。

- 改善アクションがない
- 達成率が高い
- 達成率が低い
- 高優先度の未完了がある
- 投資可能額が不足している
- 経費率が高め

表現は「影響している可能性」「一緒に確認する」「見直し候補」のようにし、断定しません。

## ホームカードとの連動

ホーム画面では「改善進捗」カードとして表示します。

- 達成率
- 未完了件数
- 完了件数
- 高優先度未完了件数
- 投資可能額
- 経費率
- 上位インサイト1件

操作は会計タブに集約し、ホームでは確認のみ行います。

## AI分析JSONとの整合性

ホームAI分析コピーには `improvementProgress` を含めます。

同じ `buildImprovementProgressReport` を使うため、会計タブ、ホーム、AI分析JSONで進捗とKPIが一致します。

## JSONエクスポートとの整合性

`buildAccountingJson(entries, month, actions)` でも `improvementProgress` を含めます。
